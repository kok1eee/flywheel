# Plan Review Checklist Reference

> このファイルは `agents/critic.md` から段階的開示（Progressive Disclosure）で分離されたリファレンスです。
> エージェント実行時に `Read` して適用してください。

## レビュー観点

### 1. 完全性
- [ ] 全ての要件がカバーされているか
- [ ] 必要なタスクが漏れていないか
- [ ] 成功基準が明確か

### 2. 実現可能性
- [ ] 技術的に実現可能か
- [ ] リソース（時間、スキル）は足りるか
- [ ] 依存関係は解決可能か

### 3. リスク管理
- [ ] 主要リスクが特定されているか
- [ ] 対策は現実的か
- [ ] フォールバックプランはあるか

### 4. 明確性
- [ ] 曖昧な表現はないか
- [ ] 各タスクの完了条件は明確か
- [ ] 優先度は設定されているか

## レビューレポートフォーマット

```markdown
# 計画レビュー: [タスク名]

## 総合評価
承認 / 条件付き承認 / 要修正

## 良い点
- [計画の強み]

## 指摘事項

### Critical（修正必須）
- **[問題]**: [説明]
  - 推奨: [修正案]

### Warning（推奨修正）
- **[問題]**: [説明]
  - 推奨: [修正案]

### Suggestion（提案）
- [改善提案]

## 見落とされたリスク
| リスク | 影響 | 提案 |
|--------|------|------|
| [リスク] | [影響] | [対策] |

## 結論
[最終的な判定と次のステップ]
```
